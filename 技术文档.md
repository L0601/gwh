# 编辑接单台账技术文档

## 1. 技术栈

当前项目采用以下技术栈：

- 前端：Vue 3 + TypeScript + Pinia + Vue Router
- 桌面容器：Tauri 2
- 后端本地能力：Rust
- 数据库：SQLite（通过 `rusqlite` 直接访问）
- 数据导入导出：`csv`

项目目标是做一款本地桌面应用，因此前端负责界面和交互，Rust 负责数据库、本地文件处理和 Tauri 命令。

## 2. 项目目录结构

当前主要目录如下：

```text
gwh/
```

## 3. 前端结构说明

## 3.1 路由

当前有 3 个主页面：

- `src/views/TaskView.vue`
- `src/views/ContactView.vue`
- `src/views/StatsView.vue`

对应功能分别是：

- 任务管理
- 对接人管理
- 统计报表

## 3.2 组件

当前最核心的可复用组件是：

- `src/components/TaskForm.vue`

它负责：

- 新建任务表单
- 编辑任务表单
- 表单校验
- 状态联动

当前该组件被 `TaskView.vue` 放到右侧抽屉中使用。

## 3.3 状态管理

主要状态集中在：

- `src/stores/task.ts`

这个 store 当前负责：

- 联系人列表
- 任务列表
- 统计结果
- 当前筛选条件
- 保存任务后的任务回查逻辑

当前 `submitTask` 的实现不是简单返回保存结果，而是：

1. 先调用后端保存
2. 再刷新任务列表
3. 如果后端返回值为空，则从最新任务列表里回查对应任务

这样做是为了尽量避免前端在个别情况下拿不到返回对象时直接报错。

## 4. 前后端交互方式

前端通过 Tauri `invoke` 调用 Rust command。

封装位置：

- `src/api.ts`

当前主要命令如下：

- `list_contacts`
- `create_contact`
- `delete_contact`
- `delete_task`
- `list_tasks`
- `save_task`
- `stats_tasks`
- `import_tasks`
- `export_tasks`

其中前端的“下载模板”目前已经不走 Rust 命令，而是直接在前端生成 CSV 下载。

原因是：

- 这条链路更简单
- 不依赖额外本地命令
- 对用户来说更稳定

## 5. Rust 后端结构说明

主要文件：

- `src-tauri/src/lib.rs`
- `src-tauri/src/db.rs`

### 5.1 `lib.rs`

职责：

- 注册 Tauri command
- 初始化应用数据目录
- 创建 SQLite 文件
- 启动 Tauri Builder

### 5.2 `db.rs`

职责：

- 定义数据结构
- 初始化 SQLite 表
- 提供联系人 CRUD
- 提供任务保存、删除、查询
- 提供统计计算
- 提供 CSV 导入导出

当前 `AppState` 内部通过：

```rust
Mutex<Connection>
```

来串行访问 SQLite。

这样实现简单直接，适合当前本地单用户桌面应用。

## 6. 数据模型

## 6.1 contacts

联系人表字段如下：

- `id`
- `name`
- `remark`
- `created_at`

说明：

- `name` 唯一
- `created_at` 当前保留到分钟

## 6.2 editing_tasks

任务表字段如下：

- `id`
- `title`
- `contact_id`
- `received_date`
- `completed_date`
- `fee`
- `print_fee`
- `settled_amount`
- `is_settled`
- `is_completed`
- `page_count`
- `word_count`

说明：

- 状态不单独存字符串
- 列表展示状态由 `is_completed + is_settled` 推导得到

当前状态映射规则：

- `is_settled = true` -> `SETTLED`
- `is_completed = true && is_settled = false` -> `COMPLETED_UNSETTLED`
- 其他 -> `IN_PROGRESS`

## 7. 当前页面实现逻辑

## 7.1 任务管理页

主要文件：

- `src/views/TaskView.vue`

当前逻辑：

- 顶部展示概览卡
- 筛选区控制 `store.filters`
- 任务列表按状态分成三栏
- 任务表单通过右侧抽屉打开
- 快捷按钮支持：
  - 标记完成
  - 标记结账
  - 编辑任务
  - 删除任务

当前样式特点：

- 三栏使用固定栏内滚动，避免任务一多页面无限变长
- 任务卡片做了紧凑化处理

## 7.2 对接人页

主要文件：

- `src/views/ContactView.vue`

当前逻辑：

- 展示对接人概览卡
- 支持新增联系人
- 支持删除联系人
- 删除前确认
- 如果有任务关联，后端拒绝删除

## 7.3 统计页

主要文件：

- `src/views/StatsView.vue`

当前逻辑：

- 按日期和对接人筛选
- 展示 3 张概览卡
- 明细区固定高度并可滚动

当前统计实现仍属于基础版，尚未引入趋势图。

## 8. 表单校验与交互规则

表单组件：

- `src/components/TaskForm.vue`

当前已实现的校验包括：

- 书名必填
- 对接人必选
- 接单日期必填
- 完成日期不能早于接单日期
- 已结账时必须填写结账金额
- 金额字段必须为非负数
- 页码、字数字段必须为非负整数

当前状态联动包括：

- 勾选“已结账”时自动同步为“已完成”
- 标记为已完成时自动带入当天完成日期
- 取消完成且未结账时，清空完成日期

## 9. 导入导出说明

## 9.1 导入

后端方法：

- `AppState::import_tasks`

当前导入格式基于 `TaskImportRecord`，字段包括：

- `title`
- `contact_name`
- `received_date`
- `completed_date`
- `fee`
- `print_fee`
- `settled_amount`
- `is_settled`
- `is_completed`
- `page_count`
- `word_count`

当前导入特性：

- 如果联系人不存在，会自动创建联系人
- 如果 CSV 行格式错误，会提示第几行出错

## 9.2 导出

后端方法：

- `AppState::export_tasks`

当前导出是把全部任务按 CSV 写出。

## 9.3 模板下载

当前模板下载由前端直接生成示例 CSV 文件。

实现位置：

- `src/views/TaskView.vue`

原因：

- 逻辑简单
- 稳定性更高
- 不依赖额外后端命令
